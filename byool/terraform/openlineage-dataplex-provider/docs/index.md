# OpenLineage Provider

The OpenLineage provider lets you declare data pipeline lineage as Terraform infrastructure.
Each `openlineage_job` resource emits an OpenLineage `COMPLETE` RunEvent to the
[GCP Dataplex Lineage API](https://cloud.google.com/data-catalog/docs/concepts/about-data-lineage),
creating a tracked Process and Run in Dataplex.

## Why declare lineage in Terraform?

- **Infrastructure-as-code** — lineage metadata lives alongside your pipeline definitions
- **Drift detection** — Terraform detects when a Dataplex Process is deleted externally and re-creates it
- **Idempotent** — re-applying emits a new Run under the same Process
- **Auditable** — changes to lineage are tracked in version control

## Authentication

The provider uses the GCP Dataplex Lineage API. Authenticate via one of:

**Application Default Credentials (recommended):**
```shell
gcloud auth application-default login
```

**Service account key file:**
```hcl
provider "openlineage" {
  credentials_file = "/path/to/service-account.json"
}
```

**Environment variable:**
```shell
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json
```

## Required GCP Permissions

The service account or user needs:
- `datalineage.processes.create`
- `datalineage.processes.get`
- `datalineage.processes.delete`
- `datalineage.runs.list`
- `datalineage.lineageEvents.create`

Or the predefined role: `roles/datalineage.admin`

## Example Usage

```terraform
terraform {
  required_providers {
    openlineage = {
      source  = "registry.terraform.io/tomasznazarewicz/openlineage"
      version = "~> 0.1"
    }
  }
}

# ──────────────────────────────────────────────────────────────────────────────
# Minimal configuration
# Uses Application Default Credentials (ADC).
# Run: gcloud auth application-default login
# ──────────────────────────────────────────────────────────────────────────────
provider "openlineage" {
  project_id = "my-gcp-project"
  region     = "us-central1"
}

# ──────────────────────────────────────────────────────────────────────────────
# With explicit service-account key file
# ──────────────────────────────────────────────────────────────────────────────
# provider "openlineage" {
#   project_id       = "my-gcp-project"
#   region           = "us-central1"
#   credentials_file = "/path/to/service-account.json"
# }

# ──────────────────────────────────────────────────────────────────────────────
# With warn_on_unused_facets disabled (useful when migrating from another
# OpenLineage consumer — suppresses warnings about unsupported facets while
# you clean up config incrementally)
# ──────────────────────────────────────────────────────────────────────────────
# provider "openlineage" {
#   project_id          = "my-gcp-project"
#   region              = "us-central1"
#   warn_on_unused_facets = false
# }
```

## Schema

### Required

- `project_id` (String) GCP Project ID for lineage emissions
- `region` (String) GCP Region for lineage API

### Optional

- `credentials_file` (String) Path to GCP credentials JSON file. If not set, will use Application Default Credentials
- `warn_on_unused_facets` (Boolean) Emit warnings when config defines facets the active consumer does not support. Defaults to true. Set to false when migrating from another consumer.
